# Blog Writing Reference

Quick-reference for creating new blog posts on anas.semesmieh.com.
Read this before writing — it replaces scanning every existing post.

## File Structure

- Posts: `blog/posts/{kebab-case-slug}.html`
- Template: `blog/posts/_post-template.html` (skeleton only — use a recent post as the real reference)
- Index: `blog/index.html` (card grid with JS search/sort/filter)
- Categories: `blog/categories/index.html`
- Tags: `blog/tags/index.html`
- RSS: `blog/rss.xml`

## Creating a New Post — Checklist

1. Create `blog/posts/{slug}.html` using the patterns below
2. Add a post card to `blog/index.html` (newest first in the grid)
3. Add the post to `blog/categories/index.html` under the right category
4. Update series-nav in related posts (if part of a series)
5. Update `blog/rss.xml` with new entry
6. Commit and push

## HTML Structure

Every post is a **fully self-contained HTML file** — no external CSS, no JS frameworks, no build tooling.

### Head Section

```html
<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>{Post Title} | Anas Semesmieh</title>
  <meta name="description" content="{≤160 chars, includes primary keywords}" />
  <meta name="author" content="Anas Semesmieh" />
  <link rel="canonical" href="https://anas.semesmieh.com/blog/posts/{slug}" />
  <meta property="og:type" content="article" />
  <meta property="og:title" content="{Post Title}" />
  <meta property="og:description" content="{shorter version, ≤120 chars}" />
  <meta property="og:url" content="https://anas.semesmieh.com/blog/posts/{slug}" />
  <meta property="og:image" content="https://anas.semesmieh.com/profile-wide.jpg" />
  <meta name="twitter:card" content="summary_large_image" />
  <meta name="twitter:title" content="{Post Title}" />
  <meta name="twitter:description" content="{same as og:description}" />
  <meta name="twitter:image" content="https://anas.semesmieh.com/profile-wide.jpg" />
  <style>/* inline CSS — copy from latest post */</style>
  <script type="application/ld+json">
  {
    "@context":"https://schema.org",
    "@type":"BlogPosting",
    "headline":"{Post Title}",
    "datePublished":"{YYYY-MM-DD}",
    "dateModified":"{YYYY-MM-DD}",
    "author":{"@type":"Person","name":"Anas Semesmieh","url":"https://anas.semesmieh.com/about/"},
    "publisher":{"@type":"Person","name":"Anas Semesmieh"},

    "image":"https://anas.semesmieh.com/profile-wide.jpg",

  }
  </script>

```

### Body Structure

```html
<body>
  <div class="wrap">
    <nav class="nav" aria-label="Post navigation">
      <a href="/">Portfolio</a>
      <a href="/blog/">All Posts</a>
      <a href="/blog/categories/">Categories</a>
      <a href="/blog/tags/">Tags</a>
      <a href="/blog/rss.xml">RSS</a>
    </nav>
    <article>
      <h1>{Title}</h1>
      <p class="meta">By <a href="/about/">Anas Semesmieh</a> &middot; {Date} &middot; {Tag1}, {Tag2}</p>
      <!-- Content -->
      <div class="series-nav"><!-- if part of a series --></div>
    </article>
  </div>
</body>
```

### Nav Links

Use 5 items: Portfolio, All Posts, Categories, Tags, RSS.

## Content Components

### Callout (max 1–2 per post)
```html
<div class="callout">
  Key insight or link to source code. Never for warnings/errors.
</div>
```

### Code Block
```html
<pre><code># Comment explaining what this does
command --flag value</code></pre>
```
No syntax highlighting classes. Use `&amp;` for `&`, `&lt;` for `<`, `&gt;` for `>`.

### Inline Code
```html
<code>variable_name</code>
```

### Table
```html
<table>
  <thead><tr><th>Column</th><th>Column</th></tr></thead>
  <tbody><tr><td>Value</td><td>Value</td></tr></tbody>
</table>
```

### Series Navigation (at bottom of `<article>`, inside it)
```html
<div class="series-nav">
  <strong>Series Name:</strong>
  <a href="/blog/posts/slug">Post Title</a>
  <a href="/blog/posts/current-slug"><strong>Current Post (this post)</strong></a>
</div>
```

## Blog Index Card

Add inside `<div class="grid" id="postGrid">`, ordered newest-first:

```html
<article data-date="{YYYY-MM-DD}" data-tags="{Tag1,Tag2,Tag3}">
  <a class="post" href="/blog/posts/{slug}">
    <h2>{Title}</h2>
    <p class="meta">{Month DD, YYYY} · {N} min read</p>
    <p>{Description}</p>
    <div class="tags">
      <span class="tag">{Tag1}</span>
      <span class="tag">{Tag2}</span>
    </div>
  </a>
</article>
```

## Writing Style

- **Tone:** First-person, direct, practitioner voice. Confident but not arrogant.
- **Opening:** Always a concrete scenario or problem statement. Never a definition or abstract concept. 1–3 sentences.
- **Sections:** 1–3 paragraphs per h2/h3, followed by code block or list. Short and scannable.
- **Heading hierarchy:** Single h1 (title). h2 for major sections. h3 for sub-topics.
- **Code pattern:** Config → verification command → output. Always show how to confirm it worked.
- **Lists:** `<strong>Item</strong> — description` pattern for bold-lead bullets.
- **Closing:** Brief reflection, 1–2 paragraphs. No summary of what was covered.

## SEO Checklist

- [ ] `<title>` follows `Post Title | Anas Semesmieh`
- [ ] `<meta name="description">` ≤160 chars with primary keywords
- [ ] `<link rel="canonical">` with full URL (no trailing slash)
- [ ] OG tags: type, title, description, url, image
- [ ] Twitter tags: summary_large_image card, title, description, image
- [ ] JSON-LD BlogPosting schema with all required fields
- [ ] Semantic heading hierarchy (single h1, h2s, h3s)
- [ ] Internal links to related posts
- [ ] Post added to blog index, categories, and RSS
